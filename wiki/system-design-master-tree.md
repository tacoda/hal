# System Design Master Tree

A topic map (Neo Kim, systemdesign.one) laying out ~50 system-design concepts as a
progression, grouped roughly by theme:

- **Qualities:** scalability, availability, reliability, latency, throughput.
- **Data stores:** database, SQL vs NoSQL, load balancing, caching, cache invalidation.
- **APIs:** API design, REST, GraphQL, gRPC, authentication.
- **Resilience & consistency:** fault tolerance, high availability, CAP theorem,
  consistency models, replication.
- **Scale techniques:** erasure coding, consensus, leader election, secrets management,
  RBAC; sharding, indexing, denormalization, ACID, BASE.
- **Messaging:** event-driven, message queue, pub/sub, sync vs async, idempotency.
- **Reliability patterns:** bulkhead, retry logic, timeout, service discovery, API
  gateway.
- **Delivery & ops:** blue-green deployment, canary release, feature flags,
  observability, logging; correlation ID, monitoring, alerting, full-text search,
  time series.

A study checklist / reference index rather than a single argument.
