# The Mother of All Architecture Diagrams

A grid (Ravi Sunkara) framing one universal architecture blueprint across five eras —
Client-Server (1990s) → 3-Tier Web (2000s) → Cloud-Native (2010s) → AI Systems (2020s)
→ Agentic AI (2026+) → Autonomous Enterprise (future) — arguing "different eras, same
principles, one architecture."

Ten layers, each shown evolving across the eras:

1. **Client / User** — from terminal to desktop → browser → mobile → chat/voice → AI agent.
2. **Experience** — how users interact, increasingly conversational and intent-capturing.
3. **Network & Edge** — routing/security/optimization.
4. **Application / Orchestration** — monolith → 3-tier → microservices → AI workflow/multi-agent orchestration.
5. **Intelligence** — business rules → rule engine → ML models → LLMs/foundation models → reasoning & planning ("the only layer that has radically evolved").
6. **Data & Knowledge** — RDBMS → NoSQL → data lake → vector DB / knowledge graph → memory/knowledge graph.
7. **Infrastructure / Compute** — on-prem → VMs → cloud (IaaS) → containers/Kubernetes → AI compute (GPUs/TPUs).
8. **Integration / External Systems** — file transfer → web services/SOAP → REST APIs → event streaming/queues → tool use / enterprise APIs.
9. **Security & Trust** — evolving across all layers.
10. **Observability & Feedback** — logs → metrics → traces → alerts → dashboards → evaluation (AI quality) / human-in-the-loop.

Laws of architecture (bottom): every system starts with user intent; logic always sits
between users and data; data is the fuel; business outcomes are the destination.
