# Four Files to Save You Two Hours a Day

A setup (MJ Jaindl) built on four markdown files that turn a generic chatbot into a
self-improving AI workforce.

| File | Role | Why it matters |
|---|---|---|
| `Agents.md` | AI org chart — each agent has a role, personality, and clear boundaries (Exec Assistant, Marketing, Ops…) | Without it your AI is a generalist; with it you have specialists that stay in their lane |
| `Context.md` | Company brain — loads every session with overview, voice, decisions, tools | Turns a generic chatbot into *your* team member; no starting from scratch |
| `Memory.md` | What the AI learns — preferences, corrections, wins, mistakes; updates itself each session | One session becomes 100 sessions of compounding knowledge |
| `Skills.md` | Reusable playbooks — draft posts, triage inbox, pull reports; write the steps once, any agent runs them | SOPs die in Google Docs; Skills run on autopilot |

**How they work together:** Context loads first (the foundation) → Agents read it and
know their role → Agents run Skills to do real work → Memory captures what happened →
next session starts smarter than the last. That's the loop.
