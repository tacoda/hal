# Coding Agents Are Hot — Notes from AI Engineer NYC

Patrick Debois's short field note from the AI Engineer NYC Summit. The one-line trend:
**last year everyone was selling a vector database; this year evals are the rage.**

What was on show:

- **Sourcegraph & Booking.com** — migration automation and review systems
- **Datadog** — SRE/DevOps agent announcements alongside evals work
- **Daytona** — a sandbox for agent code generation
- **Gitpod** — development-environment automation
- **Ellipsis Development** — a PR-review system with agent integration
- **Windsurf** — advances in autonomous coding agents

The read: the market is still early-stage despite rapid innovation, but the shift from
vector databases to **evals** signals a maturing industry. The field has moved past *"can
AI code?"* to *"how do we know if the code is good?"* — which is precisely the question
[evals / LLM-as-a-judge](evals-llm-as-a-judge.md) exists to answer, and the same
quality-verification concern behind [comprehension debt](comprehension-debt.md). The
sandbox and dev-environment tooling maps to [execution sandboxing](execution-sandboxing.md).
