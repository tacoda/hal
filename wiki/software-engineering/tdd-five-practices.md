# TDD and Its Five Supporting Practices

A puzzle-wheel diagram with **Test-Driven Development** at the center and five
interlocking practices around it:

- **Refactoring** — improve internal structure without changing behavior.
- **Modular Architecture** — clear boundaries that make change local.
- **Continuous Delivery** — always-shippable, integrated frequently.
- **Specification by Example** — requirements expressed as concrete examples/tests.
- **AI-Assisted Code Craft** — using AI within the loop as another authoring tool.

The framing: TDD is the hub, and the surrounding practices reinforce it (and each other).
